# TE-Learn Project Guidelines

## This Project

### Content Conventions

- **Content lives at repo root** — each top-level folder is a learning group
- **One file per EO** — the atomic unit of content
- **Frontmatter is required** — see `_build/templates/eo-template.md` for schema
- **uid = filename** (sans .md) — kebab-case, human-readable slug
- **eo_id/to_id in frontmatter only** — folder names are human-readable, never codes
- **Images** go in `media/` subfolder within the TO folder
- **Status field**: `draft | review | published`
- **order field**: integer controlling sequence in auto-generated TOC

### Slug Limits

- **Folders**: max 2 words (kebab-case)
- **Files**: max 4 words (kebab-case)

### Folder Naming

Group and TO folders use human-readable kebab-case names, not ID codes:
- `getting-started/` not `INTRO/`
- `dax-authoring/` not `DAX/`

### _index.md Files

- Auto-generated by `_build/scripts/generate_toc.py` — do not edit the body
- Frontmatter (uid, title, description, order) is manually maintained
- Run `python _build/scripts/generate_toc.py --write` after adding/removing EOs

### Exercise Files

- Prefer TMDL format (separate files per table/measure) over model.bim
- Each exercise has a `.gitignore` excluding `.pbi/cache.abf` and `.pbi/localSettings.json`
- `exercises/starter/` = what learner opens; `exercises/solutions/` = expected end state

### Repo Layout

```
getting-started/          Content groups at root (the star of the show)
dax-authoring/
model-maintenance/
scripting-automation/
exercises/                Hands-on exercise models
_build/                   Tooling (editors can ignore)
  scripts/                Publishing and TOC generation
  templates/              EO template
  tests/                  Content convention tests
```

### Publishing

- Primary target: EasyGenerator (no API — use Playwright automation in `_build/scripts/`)
- Secondary target: HubSpot (no KB API — manual or browser automation)
- Run: `python _build/scripts/publish_to_easygenerator.py --eo <uid>`

### Commands

| Task | Command |
|------|---------|
| Generate TOC | `python _build/scripts/generate_toc.py --write` |
| Run tests | `python -m pytest _build/tests/` |
| Publish single EO | `python _build/scripts/publish_to_easygenerator.py --eo <uid>` |
| List unpublished | `grep -rl "status: draft" getting-started/ dax-authoring/ model-maintenance/ scripting-automation/` |

## Environment

- **OS**: Windows. Use Git Bash with forward slashes.
- **Python**: Use venv if adding dependencies.
